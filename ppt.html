Arsitektur Ekosistem IoT

1. Edge (Pengirim)

Akuisisi & Analisa

Sensor memindai lingkungan. ESP32 menghitung Rate of Change air secara lokal. Data ditampilkan real-time di layar OLED lokal disertai Buzzer darurat.

2. Transport

MQTT HiveMQ

Distribusi payload data sangat ringan. Memastikan transmisi kecepatan tinggi dan tanpa packet loss menuju berbagai penerima secara bersamaan.

3. Node Aktuator & Web

Respon Otomatis

ESP32 Penerima menggerakkan Motor Servo (Gerbang Pintu Air), sementara Web Dashboard merender grafik tren dan menembak API Telegram Bot.

Pembaruan Instrumen Perangkat Keras

  • Layar OLED SSD1306 (TX): Indikator visual I2C untuk pemantauan langsung oleh warga di lokasi tanpa koneksi internet.
  • Alarm Buzzer (TX): Peringatan auditori berdesibel tinggi yang aktif berdasarkan eskalasi status bahaya.
  • Motor Servo (RX): Representasi aktuator cerdas yang membuka/menutup gerbang air secara otonom.
  • Sensor HC-SR04 & LDR: Pembaca parameter utama dengan suplai tegangan 5V stabil.

Non-Blocking Delay

Arsitektur kode menggunakan fungsi millis() murni untuk *multitasking*, memungkinkan ESP32 mengolah data lokal, dan menggerakkan aktuator tanpa memutus jaringan komunikasi nirkabel.

Algoritma Logika Keputusan Berjenjang

Kondisi Pembacaan Sensor Klasifikasi Status Aksi Aktuator Servo Notifikasi & Alarm
Jarak > 50 cm AMAN Pintu Tertutup (0°) Sistem Siaga (Senyap)
Jarak < 50 cm, Cahaya ≥ 30% WASPADA (SIANG) Buka Parsial (45°) Buzzer Beep Lambat

Full-Stack Landing Page & Dashboard

Type-Safe Architecture

Aplikasi web dibangun kokoh menggunakan TypeScript. Kami secara ketat mengeliminasi kerentanan type: any pada halaman page.tsx dengan mendefinisikan *interface* khusus untuk menangkap payload data sensor dari MQTT.

Real-Time Line Charting

Mengintegrasikan pustaka Recharts untuk merender fluktuasi grafik elevasi air. Data historis (Time-Series) membantu analis melihat pola tren air secara instan di Web Browser.

Notifikasi Push Telegram & Keamanan

Pengamanan Token & Anti-Spam

Sistem notifikasi dikelola penuh dari arsitektur Serverless Backend (Next.js API Route) untuk mencegah eksposisi rahasia Bot Token ke sisi publik/klien.

Otomatisasi Darurat

Notifikasi hanya dikirim saat sensor mendeteksi ambang batas SIAGA 1 atau potensi BANJIR BANDANG, menyertakan data level air saat itu dan anjuran evakuasi warga.

Throttle Logic Filter

Memanfaatkan fungsionalitas useRef pada React untuk memberikan jeda aman pengiriman pesan (60 detik antar-peringatan), sehingga grup WhatsApp/Telegram terhindar dari tumpukan pesan sampah.

Mitigasi Anomali Sensor (Stilling Well)

Tantangan Industri Nyata

Sensor Ultrasonik rawan mengalami False Positive jika mendeteksi objek asing di atas sungai (seperti sampah plastik atau fauna) yang menyebabkan pembacaan jarak anjlok seolah-olah terjadi banjir bandang.

Hardware: Stilling Well

Sensor tidak digantung bebas, melainkan dimasukkan ke dalam instalasi Pipa Pelindung (Stilling Well) berpori. Air sungai masuk dengan tenang, sementara sampah padat tersaring di luar area pantul sensor.

Software: Median Filtering

Pengembangan *firmware* menerapkan teknik Debounce Delay. ESP32 wajib mengambil rata-rata 5 sampel berkelanjutan sebelum mengesahkan status darurat, mengabaikan objek asing yang melintas cepat.
